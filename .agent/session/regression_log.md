## #2 [2026-04-12] Fix Text Displacement Formula and Double Transformation — Session 9d60fe32

### Attempted Fix
- Discovered and fixed issue where CTM (Cumulative Transformation Matrix) was being double-applied in SDK and Renderer
- Changed byte-unit calculation in `compute_offset` to integer calculation
- Repaired regression in glyph metrics logic

### Regression Caused
- FontMatrix inheritance broke because the Renderer was changed to use only `view_transform`
- 1000x scale error occurred in Type 0 fonts

### Learned Invariant
- The Renderer should not be designed with the assumption that CTM is baked into the glyph paths generated by the SDK
- `compute_offset` needs to treat byte sequences as a single u32 integer

## #3 [2026-04-12] Coordinate System Synchronization and FontMatrix Inheritance — Session e8855a35

### Attempted Fix
- Reflected Y-axis inversion in the Renderer's CTM and changed the SDK side to extract glyphs with `-fs`
- Unified the default FontMatrix to `[0.001, 0, 0, 0.001, 0, 0]`
- Fixed Type 0 fonts to inherit the FontMatrix of child CIDFonts

### Regression Caused
- The scaling order of FontMatrix and font_size (Tfs) became unclear
- The boundary of scale responsibility between SDK and Renderer remains ambiguous

### Cumulative Invariants

- **Coordinate System**: Transformation between PDF space (Bottom-Up) and Vello space (Top-Down) must always pass through the Document-level CTM.
- **Scaling**: Scaling of `FontMatrix` (0.001) must be performed **after** applying the PDF `W` array.
- **Verification (NEW)**: A "completion report" without quantitative evidence from automated tests risks hiding future regressions. Fixes must always be accompanied by tests.

### Learned Invariant
- FontMatrix is inherited in the Type 0 -> CIDFont parent-child relationship
- If the responsibility for the entire scaling chain (FontMatrix -> Tfs -> CTM) is not documented, fixes will be based on different assumptions every time

## Invariants Summary

> Below is a summary of invariants extracted from all entries. Keep this even if the log grows large.

1. Fixing CMaps alone does not resolve text rendering issues (advancement calculation is also involved)

4. FontMatrix is inherited from Type 0 to CIDFont
