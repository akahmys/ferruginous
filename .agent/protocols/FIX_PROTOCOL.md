# 修正プロトコル (Fix Protocol)

> [!IMPORTANT]
> 修正は開発ではない。「コードを変更した」と「問題が解決した」は同義ではない。
> 科学的方法に基づき、**診断 → 仮説検証 → 最小変更** のサイクルで進めよ。

## 1. 認識論的謙虚さ (Epistemic Humility)

- **規約**: AI は自身の判断の確信度を過大評価する傾向を自覚せよ。完了の宣言は主観的確信ではなく、客観的証拠（テスト通過）によってのみ行え。
- **目的**: 虚偽の完了報告とリグレッションの連鎖を防止する。
- **判定基準**: `.agent/session/walkthrough.md` にテスト出力のエビデンスが添付されていること。

## 2. 診断先行 (Diagnosis Before Code)

- **規約**: プロダクションロジックを変更する前に、必ず原因を診断せよ。仮説は**複数**立て、1つに固執しないこと。
- **目的**: 「推測による修正 → 別の場所が壊れる」という連鎖を断ち切る。
- **判定基準**: `.agent/session/task.md` に複数の仮説と、それを検証するための診断手順が記載されていること。

## 3. スコープの自己制限 (Bounded Scope)

- **規約**: 1つのセッションで解決できる範囲には限界がある。複雑な問題に直面した場合、検証可能な最小単位に分解し、各単位の完了を証明してから次に進め。
- **目的**: 影響範囲の制御不能を防止する。
- **判定基準**: `.agent/session/task.md` の各ステップが単独で検証可能な粒度であること。

## 4. リグレッション意識 (Regression Awareness)

- **規約**: 修正が他の箇所に与える影響を常に確認せよ。修正後に既存テストを全て実行し、新たな失敗がないことを証明せよ。
- **目的**: 「モグラ叩き」パターンの根絶。
- **判定基準**: `cargo test` の全テスト PASS、および `.agent/session/regression_log.md` への「リグレッションなし」の明示的記録。

---

## 5. 修正サイクル (Fix Cycle)

以下の手順を厳守せよ。ステップの省略は禁止。

```
1. 現象の記述  ──→ 何が起きているか正確に .agent/session/task.md に記録する
2. 履歴の確認  ──→ .agent/session/regression_log.md から過去の試行を読む
3. 仮説の列挙  ──→ なぜ起きているか、候補を複数挙げる（1つに絞らない）
4. 診断の設計  ──→ プロダクションロジックを変更せずに仮説を検証する方法を設計
5. 診断の実行  ──→ ログ・テストで仮説を絞り込む
6. 最小変更    ──→ 1つだけ変更を加える
7. 効果の検証  ──→ 症状が消えたか確認する
8. リグレッション確認 ──→ cargo test で他の箇所が壊れていないか確認する
9. 記録       ──→ 結果を .agent/session/task.md, .agent/session/regression_log.md に記録する
```

### ステップ 4: 診断で許される変更の範囲

- ✅ OK: ログ出力の追加（`eprintln!` 等）
- ✅ OK: 新規テストの追加（既存コードを呼び出して値を検証する）
- ✅ OK: テスト用の診断バイナリ・スクリプトの追加
- ❌ NG: プロダクションロジックの変更（関数のロジック書き換え）
- ❌ NG: 変換パイプラインの構造変更

**基準**: プロダクションロジックの挙動を変更しない。テストやログは「観測行為」であり、系を乱さない。

---

## 6. 完了ゲート (Completion Gate)

修正を「完了」と宣言するには、以下の **すべて** を満たすこと。

### A. リグレッションテスト [MUST]

- 修正対象のバグに対するテストが test suite に存在すること
- そのテストが `cargo test` で PASS すること

### B. 既存テスト全通過 [MUST]

- `cargo test` で全テストが PASS すること（リグレッションなし）

### C. エビデンスの記録 [MUST]

- テスト出力のログが `.agent/session/walkthrough.md` に添付されていること
- `.agent/session/regression_log.md` に「リグレッションなし」が明記されていること

### D. 視覚的エビデンス（UI 変更時のみ）[MUST]

- スクリーンショットまたは録画が `.agent/session/walkthrough.md` に添付されていること

### E. コンプライアンス [SHOULD]

- `verify_compliance.sh` が PASS すること

### 禁止事項

- 自動テスト無しに「完了」と宣言してはならない
- 「目視で確認しました」のみでは完了とみなさない
- ROADMAP の [完了] マーカーは、本ゲートを通過し**ユーザー承認を得た**場合のみ付与してよい
